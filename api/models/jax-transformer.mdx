## Overview

The JAX implementation provides a **pure functional** scaled dot-product attention mechanism. Unlike the PyTorch version, this is a low-level building block rather than a complete model, demonstrating JAX's functional programming approach.

<Note>
  This implementation focuses on the **core attention mechanism** only. It's designed for research and custom implementations where you need fine-grained control over the attention computation.
</Note>

## Function Definition

### scaled_dot_product_attention_jax

```python
def scaled_dot_product_attention_jax(
    query_matrix: Any,
    key_matrix: Any,
    value_matrix: Any,
) -> Any:
```

Computes scaled dot-product attention using JAX array operations.

**Source**: `ai_researcher_repo/models/transformer_jax.py:14`

</ResponseField>

## Algorithm Implementation

The function implements the standard scaled dot-product attention formula:

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

### Step-by-Step Breakdown

```python
# 1. Extract key dimension for scaling
key_dimension = query_matrix.shape[-1]
scaling_factor = 1.0 / math.sqrt(key_dimension)
```

The scaling factor is $\frac{1}{\sqrt{d_k}}$ where $d_k$ is the key dimension.

```python
# 2. Compute attention scores (Q @ K^T)
attention_scores = jax_numpy.matmul(
    query_matrix,
    jax_numpy.swapaxes(key_matrix, -1, -2)
)
```

Computes the dot product between queries and keys. `swapaxes` transposes the last two dimensions.

```python
# 3. Scale the scores
scaled_scores = attention_scores * scaling_factor
```

Applies the $\frac{1}{\sqrt{d_k}}$ scaling to prevent gradients from vanishing.

```python
# 4. Compute attention weights (manual softmax)
attention_weights = jax_numpy.exp(scaled_scores)
attention_weights = attention_weights / attention_weights.sum(axis=-1, keepdims=True)
```

**Manual softmax implementation** for educational clarity:
- Exponentiate the scaled scores
- Normalize by the sum across the key dimension

<Warning>
  This implementation uses a **numerically unstable softmax** for clarity. Production code should use `jax.nn.softmax` which includes the log-sum-exp trick.
</Warning>

```python
# 5. Apply attention weights to values
return jax_numpy.matmul(attention_weights, value_matrix)
```

Multiplies the attention weights by the value matrix to produce the final output.

### Multi-Head Attention Pattern

```python
import jax
import jax.numpy as jnp
from ai_researcher_repo.models.transformer_jax import (
    scaled_dot_product_attention_jax
)

# Setup
batch_size = 4
seq_len = 20
model_dim = 128
num_heads = 8
head_dim = model_dim // num_heads  # 16

# Project to Q, K, V (normally done with learned linear layers)
rng = jax.random.PRNGKey(0)
x = jax.random.normal(rng, (batch_size, seq_len, model_dim))

# Reshape for multi-head attention: (batch, num_heads, seq_len, head_dim)
q = x.reshape(batch_size, seq_len, num_heads, head_dim).transpose(0, 2, 1, 3)
k = x.reshape(batch_size, seq_len, num_heads, head_dim).transpose(0, 2, 1, 3)
v = x.reshape(batch_size, seq_len, num_heads, head_dim).transpose(0, 2, 1, 3)

print(f"Q shape: {q.shape}")  # (4, 8, 20, 16)

# Apply attention to each head (vmap over head dimension)
def attention_single_head(q_head, k_head, v_head):
    return scaled_dot_product_attention_jax(q_head, k_head, v_head)

# Vectorize over the head dimension (axis 1)
attention_output = jax.vmap(
    attention_single_head,
    in_axes=(1, 1, 1),
    out_axes=1
)(q, k, v)

print(f"Attention output shape: {attention_output.shape}")  # (4, 8, 20, 16)

# Concatenate heads back together
output = attention_output.transpose(0, 2, 1, 3).reshape(
    batch_size, seq_len, model_dim
)
print(f"Final output shape: {output.shape}")  # (4, 20, 128)
```

### Integration with JAX Transformations

```python
import jax
import jax.numpy as jnp
from ai_researcher_repo.models.transformer_jax import (
    scaled_dot_product_attention_jax
)

# Define a simple attention layer with parameters
def attention_layer(params, x):
    """Single-head attention with learned projections."""
    q = jnp.dot(x, params['W_q'])
    k = jnp.dot(x, params['W_k'])
    v = jnp.dot(x, params['W_v'])
    return scaled_dot_product_attention_jax(q, k, v)

# Initialize parameters
rng = jax.random.PRNGKey(42)
model_dim = 32
params = {
    'W_q': jax.random.normal(rng, (model_dim, model_dim)) * 0.02,
    'W_k': jax.random.normal(rng, (model_dim, model_dim)) * 0.02,
    'W_v': jax.random.normal(rng, (model_dim, model_dim)) * 0.02,
}

# Sample input
x = jax.random.normal(rng, (5, model_dim))  # 5 tokens

# Forward pass
output = attention_layer(params, x)
print(f"Output: {output.shape}")  # (5, 32)

# Compute gradients with respect to parameters
grad_fn = jax.grad(lambda p: jnp.sum(attention_layer(p, x) ** 2))
grads = grad_fn(params)
print(f"Gradient keys: {grads.keys()}")  # dict_keys(['W_q', 'W_k', 'W_v'])
```

## Design Philosophy

### Functional Purity

The function is **pure** with no side effects:
- No mutable state
- No randomness (deterministic)
- Output depends only on inputs

This enables JAX transformations like:
- `jax.grad`: Automatic differentiation
- `jax.vmap`: Automatic vectorization
- `jax.jit`: JIT compilation with XLA

### Manual Softmax

The implementation uses a **manual softmax** for educational purposes:

```python
attention_weights = jax_numpy.exp(scaled_scores)
attention_weights = attention_weights / attention_weights.sum(axis=-1, keepdims=True)
```

**Production alternative**:
```python
attention_weights = jax.nn.softmax(scaled_scores, axis=-1)
```

The production version is numerically stable and prevents overflow.

### Type Annotations

The function uses `Any` type hints:

```python
def scaled_dot_product_attention_jax(
    query_matrix: Any,
    key_matrix: Any,
    value_matrix: Any,
) -> Any:
```

This allows flexibility with different JAX array types while avoiding import-time dependencies.

## Comparison with PyTorch

| Aspect | JAX Implementation | PyTorch Implementation |
|--------|-------------------|------------------------|
| **Abstraction** | Function (attention only) | Class (full model) |
| **State** | Stateless, pure function | Stateful nn.Module |
| **Scope** | Single operation | Complete language model |
| **Parameters** | None (bring your own) | Learned embeddings, layers |
| **Masking** | Not included | Causal mask built-in |
| **Use Case** | Custom research code | Ready-to-train model |

## Performance Considerations

### XLA Compilation

For best performance, JIT-compile the function:

```python
import jax
from ai_researcher_repo.models.transformer_jax import (
    scaled_dot_product_attention_jax
)

# Compile with JIT
attention_jit = jax.jit(scaled_dot_product_attention_jax)

# First call: compilation overhead
output1 = attention_jit(query, key, value)  # Slow (compiles)

# Subsequent calls: fast!
output2 = attention_jit(query, key, value)  # Fast (cached)
```

### Memory Efficiency

The attention matrix scales as $O(n^2)$ where $n$ is sequence length. For long sequences, consider:

- **Flash Attention**: Linear memory attention
- **Chunking**: Process in smaller blocks
- **Sparse Attention**: Attend to subset of tokens
