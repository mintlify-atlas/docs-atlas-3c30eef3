## Core Concept: Gradient Synchronization

Data-parallel workers compute local gradients that must be aggregated consistently. The fundamental formula is:

```
g_avg = (1 / W) * sum(g_worker)
```

### Symbol Glossary

- **W**: number of workers
- **g_worker**: gradient from one worker
- **g_avg**: synchronized average gradient
- **theta**: model parameters

### How It Works

<Accordion title="Derivation Steps">
  1. **Collect Gradients**: Gather one gradient vector from each worker
  2. **Average Components**: Average each gradient component across all workers
  3. **Apply Update**: Use the synchronized gradient for one consistent optimizer step
  
  This ensures all model replicas remain synchronized despite computing gradients on different data batches.
</Accordion>

<Note>
  Averaging gradients across workers is mathematically equivalent to computing the gradient on the combined batch from all workers, but allows for parallel computation.
</Note>

### Example Implementation

<Accordion title="Reference Solution">
  ```python
  def average_worker_gradients(worker_gradients: list[list[float]]) -> list[float]:
      worker_count = len(worker_gradients)
      gradient_dimension = len(worker_gradients[0])
      return [
          sum(worker_gradients[worker_index][dimension_index] for worker_index in range(worker_count))
          / worker_count
          for dimension_index in range(gradient_dimension)
      ]
  ```
  
  **Key insights:**
  - Use list comprehension for clean per-dimension averaging
  - Sum across all workers for each dimension
  - Divide by worker count to get the mean
</Accordion>

### Toy Example

```python

    [0.0, 0.2]

# Dimension 0: (0.2 + 0.4 + 0.0) / 3 = 0.2
# Dimension 1: (-0.1 + 0.3 + 0.2) / 3 = 0.1333
```

The shared update keeps replicas in sync across all workers.
