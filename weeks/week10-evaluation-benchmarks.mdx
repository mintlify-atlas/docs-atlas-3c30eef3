## Core Concept: Classification Metrics

A single metric can hide failure modes. The F1 score combines precision and recall:

```
F1 = 2 * (precision * recall) / (precision + recall)
```

### Symbol Glossary

- **TP**: True positives (correctly predicted positive)
- **FP**: False positives (incorrectly predicted positive)
- **FN**: False negatives (incorrectly predicted negative)
- **TN**: True negatives (correctly predicted negative)

### Confusion Matrix Metrics

<Accordion title="Metric Derivations">
  **Precision**: What fraction of positive predictions were correct?
  ```
  precision = TP / (TP + FP)
  ```
  
  **Recall**: What fraction of actual positives were found?
  ```
  recall = TP / (TP + FN)
  ```
  
  **F1 Score**: Harmonic mean of precision and recall
  ```
  F1 = 2 * (precision * recall) / (precision + recall)
  ```
  
  **Accuracy**: Overall correctness
  ```
  accuracy = (TP + TN) / (TP + FP + FN + TN)
  ```
</Accordion>

<Note>
  F1 score is particularly useful for imbalanced datasets where accuracy alone can be misleading. A model that always predicts the majority class will have high accuracy but low F1.
</Note>

### Example Implementation

<Accordion title="Reference Solution">
  ```python
  def binary_classification_metrics(
      predictions: list[int],
      labels: list[int],
  ) -> dict[str, float]:
      true_positive = sum(
          1 for prediction, label in zip(predictions, labels) 
          if prediction == 1 and label == 1
      )
      false_positive = sum(
          1 for prediction, label in zip(predictions, labels) 
          if prediction == 1 and label == 0
      )
      false_negative = sum(
          1 for prediction, label in zip(predictions, labels) 
          if prediction == 0 and label == 1
      )
      true_negative = sum(
          1 for prediction, label in zip(predictions, labels) 
          if prediction == 0 and label == 0
      )

      precision = true_positive / (true_positive + false_positive)
      recall = true_positive / (true_positive + false_negative)
      f1_score = 2.0 * precision * recall / (precision + recall)
      accuracy = (true_positive + true_negative) / len(predictions)
      
      return {
          "precision": precision,
          "recall": recall,
          "f1_score": f1_score,
          "accuracy": accuracy,
      }
  ```
  
  **Key insights:**
  - Use generator expressions with `zip` for clean pair-wise comparison
  - Compute all confusion matrix values first
  - Calculate derived metrics from base counts
  - Return structured dictionary for easy access
</Accordion>

### Toy Example

```python
predictions = [1, 0, 1, 1, 0]
labels      = [1, 0, 0, 1, 0]

# TP = 2 (indices 0, 3)
# FP = 1 (index 2)
# FN = 0
# TN = 2 (indices 1, 4)

# Precision = 2/3 ≈ 0.667
# Recall = 2/2 = 1.0
# F1 = 2 * (0.667 * 1.0) / (0.667 + 1.0) = 0.8
# Accuracy = 4/5 = 0.8
```

<Warning>
  **Edge Case**: When TP + FP = 0 or TP + FN = 0, precision or recall becomes undefined. Consider how to handle division by zero in production code.
</Warning>

## When to Use Each Metric

<Accordion title="Metric Selection Guide">
  **Accuracy**: Use when classes are balanced and all errors are equally costly
  
  **Precision**: Use when false positives are more costly (e.g., spam detection - don't want to mark real emails as spam)
  
  **Recall**: Use when false negatives are more costly (e.g., disease detection - don't want to miss sick patients)
  
  **F1 Score**: Use when you need a balance between precision and recall, or when classes are imbalanced
</Accordion>
