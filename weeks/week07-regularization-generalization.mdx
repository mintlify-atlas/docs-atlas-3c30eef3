## Core Formula

```
x_dropout = (mask * x) / keep_probability
```

## Mathematical Intuition

Inverted dropout preserves expected activation scale while randomly removing feature pathways.

<Accordion title="Symbol Glossary">
  - **mask**: binary keep/drop indicators (1 = keep, 0 = drop)
  - **keep_probability**: fraction of units kept (e.g., 0.75 = drop 25%)
  - **x**: activation vector
  - **x_dropout**: regularized activation vector
</Accordion>

## Derivation Steps

<Steps>
  <Step title="Apply binary mask">
    Multiply activations by binary mask (randomly zero out some values).
  </Step>
  <Step title="Scale surviving activations">
    Divide by keep probability to maintain expected value.
  </Step>
  <Step title="Maintain expectations">
    Expected value across training steps stays constant.
  </Step>
</Steps>

## Why Inverted Dropout?

<Info>
**Inverted dropout** scales during training so that inference requires no changes:

- **Training**: Apply mask and scale by `1/keep_prob`
- **Inference**: Use activations as-is (no scaling needed)

This is better than standard dropout which requires scaling at inference time.
</Info>

## Toy Example Walkthrough

1. Input activations: `[2, 4, 6, 8]`
2. Mask: `[1, 0, 1, 1]` and keep probability `0.75`
3. Output: `[2/0.75, 0, 6/0.75, 8/0.75] = [2.67, 0, 8.0, 10.67]`
4. Expected value is preserved after scaling

### Implementation

```python
from __future__ import annotations

def run_toy_example() -> dict[str, float]:
    activations = [2.0, 4.0, 6.0, 8.0]
    keep_mask = [1, 0, 1, 1]
    keep_probability = 0.75
    dropped = [
        activation * mask / keep_probability
        for activation, mask in zip(activations, keep_mask)
    ]
    return {
        "primary_metric": sum(dropped) / len(dropped),
        "secondary_metric": sum(activations) / len(activations),
    }
```

<Accordion title="Reference Solution">
```python
from __future__ import annotations

def apply_inverted_dropout(
    values: list[float],
    keep_mask: list[int],
    keep_probability: float,
) -> list[float]:
    return [
        value * mask / keep_probability
        for value, mask in zip(values, keep_mask)
    ]
```
</Accordion>

## Formula to Code Mapping

- Masking maps to elementwise multiplication with binary values
- Inverted scaling maps to division by keep probability
- Result maintains expected activation scale

## Dropout Benefits

Dropout helps prevent overfitting by:

1. **Reducing co-adaptation** - neurons can't rely on specific other neurons
2. **Ensemble effect** - each forward pass is like training a different subnetwork
3. **Feature redundancy** - forces learning of robust, redundant representations
4. **Improved generalization** - typically reduces validation/test error

## When to Use Dropout

<Warning>
Modern transformers often use **less dropout** than earlier architectures:
- Attention dropout: 0.0 - 0.1
- Residual dropout: 0.0 - 0.1
- Very large models may use no dropout at all

Other regularization techniques (weight decay, gradient clipping) are often more important.
</Warning>
