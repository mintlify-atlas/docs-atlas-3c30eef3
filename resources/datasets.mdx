## Dataset Philosophy

The curriculum follows a **toy-by-default, opt-in-for-scale** approach to datasets. This ensures:
- Fast iteration during development
- Safe CI/CD without expensive compute
- Clear boundaries between learning and production-scale training

<Note>
Tests and default configurations **never** launch full-dataset training jobs. Large-scale experiments are explicit opt-in only.
</Note>

## Default Local-Safe Datasets

These datasets run locally, train quickly, and require no special setup:

### Synthetic Character Corpus

<Accordion title="Generated Locally">
  **Purpose:** Unit testing and quick iteration
  
  **Characteristics:**
  - Generated on-the-fly from code
  - Deterministic for reproducible tests
  - No download required
  - Ideal for validating tokenizer and embedding logic
  
  **Use Cases:**
  - Unit tests for tokenization
  - Embedding layer verification
  - Quick sanity checks
  
  **Example:**
  ```python
  # Synthetic character-level corpus
  synthetic_text = "abcdefghijklmnopqrstuvwxyz" * 100
  ```
</Accordion>

### Tiny Shakespeare

<Accordion title="Small Open Text Corpus">
  **Purpose:** Toy language modeling experiments
  
  **Characteristics:**
  - ~1MB text file
  - Complete works of Shakespeare
  - Character-level or BPE tokenization
  - Fast training on CPU or single GPU
  
  **Use Cases:**
  - Week 6: First training loops
  - Week 7: Regularization experiments
  - Quick architecture validation
  
  **Download:**
  ```bash
  wget https://raw.githubusercontent.com/karpathy/char-rnn/master/data/tinyshakespeare/input.txt
  ```
  
  **Training Time:** ~5-10 minutes on CPU for small models
</Accordion>

### WikiText Compact Subsets

<Accordion title="Optional Toy Experiments">
  **Purpose:** Slightly larger vocabulary and structure
  
  **Characteristics:**
  - WikiText-2 (~2MB) or WikiText-103 (~500MB)
  - Preprocessed Wikipedia articles
  - Word-level or subword tokenization
  - Still manageable on local hardware
  
  **Use Cases:**
  - Week 8: Small-scale scaling experiments
  - Week 10: Evaluation benchmark baselines
  
  **Access:**
  ```python
  from torchtext.datasets import WikiText2, WikiText103
  train_iter = WikiText2(split='train')
  ```
</Accordion>

## Data Preparation Workflow

### 1. Download and Verify

```python
import hashlib

def verify_download(filepath: str, expected_hash: str) -> bool:
    """Verify downloaded dataset integrity."""
    with open(filepath, 'rb') as f:
        file_hash = hashlib.sha256(f.read()).hexdigest()
    return file_hash == expected_hash
```

### 2. Tokenization

<Accordion title="Tokenization Pipeline">
  **Steps:**
  1. Load raw text data
  2. Train or load tokenizer (BPE, WordPiece, etc.)
  3. Tokenize corpus
  4. Save tokenized indices
  
  **Example:**
  ```python
  from tokenizers import ByteLevelBPETokenizer
  
  # Train tokenizer
  tokenizer = ByteLevelBPETokenizer()
  tokenizer.train(
      files=["data/raw/corpus.txt"],
      vocab_size=32000,
      min_frequency=2
  )
  
  # Save for reuse
  tokenizer.save_model("data/tokenizers/")
  ```
</Accordion>

### 3. Preprocessing and Chunking

<Accordion title="Efficient Data Loading">
  **Strategies:**
  - **Memory-mapped arrays:** For large datasets that don't fit in RAM
  - **Sharding:** Split data across multiple files for distributed training
  - **Streaming:** Load data on-the-fly without full download
  
  **Example:**
  ```python
  import numpy as np
  
  # Memory-mapped tokenized data
  token_ids = np.memmap(
      'data/processed/tokens.bin',
      dtype=np.uint16,
      mode='r'
  )
  
  # Chunk into sequences
  def get_batch(offset: int, batch_size: int, seq_len: int):
      indices = offset + np.arange(batch_size) * seq_len
      batch = np.stack([token_ids[i:i+seq_len] for i in indices])
      return batch
  ```
</Accordion>

### 4. Validation Split

<Accordion title="Held-Out Data">
  **Best Practices:**
  - Reserve 1-5% for validation
  - Ensure validation split is representative
  - Never touch test set during development
  
  **Example:**
  ```python
  from sklearn.model_selection import train_test_split
  
  train_data, val_data = train_test_split(
      full_dataset,
      test_size=0.05,
      random_state=42
  )
  ```
</Accordion>

## Safety Guardrails

<Note>
**Protection Mechanisms:**

1. **Configuration flags:** Large datasets require explicit `--confirm-large-dataset`
2. **Size warnings:** CLI warns before downloading >1GB
3. **Test isolation:** Tests use synthetic or tiny datasets only
4. **CI defaults:** GitHub Actions never download large data
</Note>

## Using Larger Datasets

When you're ready to scale:

1. **Verify resources:**
   - Storage: >100GB free for Pile/C4
   - Compute: Multi-GPU setup recommended
   - Time: Budget hours or days for training

2. **Enable opt-in:**
   ```bash
   # Explicit confirmation required
   python train.py \
       --dataset pile \
       --subset pubmed \
       --confirm-large-dataset \
       --save-dir checkpoints/pile-experiment
   ```

3. **Monitor progress:**
   - Use TensorBoard or Weights & Biases
   - Set checkpointing frequency
   - Validate on small subset first

4. **Document experiments:**
   - Record dataset version and subset
   - Note preprocessing steps
   - Save configuration files
