## Overview

The AI Researcher curriculum is built on a carefully curated collection of primary sources from leading institutions, research labs, and practitioners. Every reference is verified for relevance and linked to specific curriculum modules.

<Note>
All sources include a `last_verified_date` and are reviewed before module linkage updates. This ensures the curriculum remains current with the latest research and best practices.
</Note>

## Using References Effectively

**Before each week:**
- Review linked sources for that module
- Start with foundational papers, then explore courses and tutorials
- Use blog posts for visual explanations and intuition

**During implementation:**
- Reference framework documentation for API details
- Compare your implementation against reference repositories
- Cross-reference multiple sources for deeper understanding

**After completion:**
- Explore research lab indices for cutting-edge developments
- Read follow-up papers cited in foundational work

### Foundational Papers

Seminal research that shaped the field:

<Accordion title="Attention Is All You Need (2017)">
  **Citation:** Vaswani et al., 2017
  
  **URL:** https://arxiv.org/abs/1706.03762
  
  **Relevance:** The paper that introduced transformers. Essential reading for weeks 4 (attention mechanics) and 5 (transformer architecture).
  
  **Key Contributions:**
  - Multi-head self-attention mechanism
  - Positional encodings
  - Architecture that enabled parallelization
</Accordion>

<Accordion title="Language Models are Few-Shot Learners (GPT-3, 2020)">
  **Citation:** Brown et al., 2020
  
  **URL:** https://arxiv.org/abs/2005.14165
  
  **Relevance:** Demonstrated scaling laws and few-shot learning. Core for weeks 8 (scaling laws) and 10 (evaluation).
  
  **Key Contributions:**
  - Few-shot learning without fine-tuning
  - Scaling behavior analysis
  - Benchmark performance across tasks
</Accordion>

<Accordion title="Training Compute-Optimal LLMs (Chinchilla, 2022)">
  **Citation:** Hoffmann et al., 2022
  
  **URL:** https://arxiv.org/abs/2203.15556
  
  **Relevance:** Chinchilla scaling laws for compute-optimal training. Essential for week 8.
  
  **Key Contributions:**
  - Optimal model size vs. training tokens
  - Compute budget allocation
  - Redefining scaling best practices
</Accordion>
