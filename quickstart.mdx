This quickstart guide will take you from installation to running your first week's challenge in about 5-10 minutes.

## Prerequisites

- Python 3.11 or higher
- Basic terminal/command line familiarity
- Git (for cloning the repository)

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the AI Researcher Zero-to-SOTA repository to your local machine:

    ```bash
    git clone <your-repo-url>
    cd ai-researcher-repo
    ```
  </Step>

  <Step title="Create a virtual environment">
    Set up an isolated Python environment to avoid dependency conflicts:

    ```bash
    python3 -m venv .venv
    source .venv/bin/activate  # On Windows: .venv\Scripts\activate
    ```

    <Note>
      Always activate your virtual environment before working with the repository. You'll know it's active when you see `(.venv)` in your terminal prompt.
    </Note>
  </Step>

  <Step title="Install development dependencies">
    Install the package in editable mode with all development tools:

    ```bash
    pip install -e .[dev]
    ```

    This installs:
    - Core dependencies (numpy, pyyaml)
    - Development tools (pytest, ruff, mypy)
    - The `ai-researcher` CLI command
  </Step>

  <Step title="Verify installation">
    Run the baseline quality checks to ensure everything is set up correctly:

    ```bash
    make run-checks
    ```

    Or use the CLI directly:

    ```bash
    ai-researcher run-checks
    ```

    <Note>
      This command runs linting, type checking, and fast unit tests. If all checks pass, you're ready to start learning!
    </Note>
  </Step>
</Steps>

## Your First Week

Let's walk through a complete learning cycle with Week 1: Math, Optimization, and Autograd.

<Steps>
  <Step title="Navigate to Week 1">
    ```bash
    cd curriculum/weeks/week01_math_optimization_autograd
    ```

    Each week follows a consistent structure:
    - `README.md` - Module overview and learning outcomes
    - `math_walkthrough.md` - Detailed mathematical derivations
    - `toy_example.py` - Runnable demonstration code
    - `coding_problem.md` - Challenge prompt
    - `problem.py` - Your implementation file
    - `solution.py` - Reference solution
    - `tests/` - Test files to validate your work
  </Step>

  <Step title="Read the module README">
    Start by understanding the module's focus:

    ```bash
    cat README.md
    ```

    You'll learn:
    - The core formula: `L(theta) = (theta - 3)^2`
    - Learning outcomes
    - Reference materials (Stanford CS336, OpenAI Research, DeepMind)
  </Step>

  <Step title="Run the toy example">
    See gradient descent in action:

    ```bash
    python toy_example.py
    ```

    Or run it as a module:

    ```bash
    python -c "from toy_example import run_toy_example; print(run_toy_example())"
    ```

    Expected output:
    ```python
    {'primary_metric': 3.0625, 'secondary_metric': 0.00390625}
    ```
  </Step>

  <Step title="Implement your solution">
    1. Read `coding_problem.md` for the challenge description
    2. Open `problem.py` in your editor
    3. Implement the required functions
    4. Save your changes

    <Note>
      Don't peek at `solution.py` until you've attempted the problem yourself!
    </Note>
  </Step>

  <Step title="Test your implementation">
    Run the module tests to validate your solution:

    ```bash
    ai-researcher run-module-tests --week week01_math_optimization_autograd
    ```

    Or use pytest directly:

    ```bash
    pytest tests/
    ```

    <Warning>
      Exercise tests in `test_problem_exercise.py` are skipped by default. Unskip them (remove `@pytest.mark.skip`) when you're ready to test your implementation.
    </Warning>
  </Step>

  <Step title="Compare with reference solution">
    After passing tests, review the reference solution:

    ```bash
    cat solution.py
    ```

    Write a short reflection:
    - What did you learn?
    - What would you do differently?
    - What concepts need more practice?
  </Step>
</Steps>

## Quick CLI Reference

The `ai-researcher` CLI provides several useful commands:

<CodeGroup>
```bash Quality Checks
# Run all quality checks (lint, typecheck, tests)
ai-researcher run-checks

# Or use the Makefile shortcut
make run-checks
```

```bash Module Tests
# Test a specific week
ai-researcher run-module-tests --week week01_math_optimization_autograd

# Test week 4 (attention mechanics)
ai-researcher run-module-tests --week week04_attention_mechanics
```

```bash Toy Training
# Train a tiny language model with default config
ai-researcher train-toy

# Use a custom configuration
ai-researcher train-toy --config configs/defaults.yaml
```

```bash Evaluation
# Run toy evaluation metrics
ai-researcher eval
```
</CodeGroup>

## Common Issues

<Accordion title="Command not found: ai-researcher">
  Make sure you've:
  1. Activated your virtual environment: `source .venv/bin/activate`
  2. Installed the package: `pip install -e .[dev]`
  
  The CLI is registered as a console script in `pyproject.toml`.
</Accordion>

<Accordion title="Import errors when running tests">
  This usually means dependencies aren't installed. Run:
  ```bash
  pip install -e .[dev]
  ```
  
  For framework-specific code, install the appropriate extra:
  ```bash
  pip install -e .[pytorch]  # For PyTorch examples
  pip install -e .[jax]      # For JAX examples
  ```
</Accordion>

<Accordion title="Tests are failing unexpectedly">
  1. Ensure you're in the repository root directory
  2. Check that your virtual environment is activated
  3. Run `make run-checks` to verify baseline setup
  4. Check the test output for specific error messages
</Accordion>
