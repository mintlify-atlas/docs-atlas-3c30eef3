# Realm Deep-Dives

Realm deep-dives are specialized learning tracks that build on the 16-week core curriculum. Each realm focuses on a specific domain of AI research and engineering, providing project ladders, reference materials, and acceptance criteria for mastery.

## When to Use Realms

Realms are designed for learners who have:
- Completed the relevant core curriculum weeks (typically weeks 1-12)
- Gained comfort with transformer architectures and training loops
- Built a foundation in evaluation and testing practices
- Want to specialize in a specific AI domain

<Note>
You can pursue realms alongside the core curriculum or after completing all 16 weeks. Each realm is self-contained and can be studied independently.
</Note>

## How Realms Work

Each realm provides:

## Realm Workflow

Follow this workflow for each realm:

1. **Complete Prerequisites**: Finish related core weeks first
2. **Pick a Realm**: Choose based on your interests or career goals
3. **Follow the Project Ladder**: Start with Short, progress to Medium, then Deep
4. **Cross-Check with References**: Validate implementation decisions against listed sources
5. **Document Findings**: Record experiment notes and learnings before moving to the next realm

<Accordion title="Suggested Realm Progression">
  While realms can be pursued in any order, this progression builds complementary skills:
  
  1. **LLM Systems** (foundation for all other realms)
  2. **Vision Transformers** or **RL Sequence Models** (pick one modality)
  3. **Multimodal Alignment** (combines multiple modalities)
  4. **Interpretability** (understanding model internals)
  5. **Alignment & Safety** (responsible AI development)
  6. **ML Systems & Scaling** (production deployment)
</Accordion>

## Project Ladder Tiers

Each realm uses a consistent three-tier structure:

### Short Projects (1-2 days)
- Implement core concepts on toy data
- Validate mathematical understanding
- Build reusable components

### Medium Projects (3-7 days)
- Add evaluation metrics and ablations
- Train on small real datasets
- Compare design choices

### Deep Projects (1-2 weeks)
- Run controlled experiments with full methodology
- Benchmark against baselines
- Document tradeoffs and failure modes

## Acceptance Criteria

For each realm, aim to meet all acceptance criteria:

- **Reproducibility**: All experiments can be re-run with same results
- **Documentation**: Clear explanations of design choices and findings
- **Source-Backed**: Architectural decisions reference authoritative sources
- **Evaluation**: Metrics are appropriate, reported, and interpreted
- **Failure Analysis**: Known limitations and failure modes documented
